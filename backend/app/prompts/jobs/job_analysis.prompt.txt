### Critical Rules for Small Models
1. **NO Single Letters:** Do not output single letters like "c", "r", "f" unless explicitly written as "C Language" or "R Language". If unsure, ignore.
2. **Explicit Only:** If a technology (e.g., "Docker") is mentioned, put it in `tech_stack`. Do not hallucinate technologies not in the text.
3. **Tech Stack Categorization:**
   - `languages`: Python, Java, C++, C#, JavaScript, TypeScript, Go, Rust.
   - `cloud`: AWS, Azure, GCP.
   - `tools`: Git, Docker, Jenkins, Kubernetes, Terraform, Linux, Windows.
   - `management`: Agile, Scrum, CI/CD, DevOps.
   - `domains`: Cyber, Embedded, RTOS, Defense.
4. **Skills:**
   - `must_have`: Extract specific hard skills marked as required (e.g., "Experience with Python").
   - `nice_to_have`: Extract skills marked as advantage/bonus.
