### 1. SKILLS MATCH (25 points) - TECHNICAL FOUNDATION ⚡

PROCESS:
Step 1: Extract the TOP 6-8 CRITICAL technical requirements (ignore generic tools like Jira/Slack unless central to role)
Step 2: List candidate's technical skills from resume
Step 3: Count exact and similar matches
Step 4: Calculate score

FORMULA: (matched_skills / critical_skills_count) × 25

MATCHING RULES:
✓ Exact match = 1.0 credit (e.g., "Python" = "Python")
✓ Framework counts as parent language = 1.0 credit (e.g., "React" counts for "JavaScript")
✓ Similar technology = 0.5 credit (e.g., "PostgreSQL" ≈ "MySQL")
✓ **Ecosystem Transferability** = 0.8 credit. Related frameworks in same ecosystem (e.g., React Web ↔ React Native, Java Spring ↔ Java EE).
✓ **Transferable Skills (Tech Agility)** = 0.7 credit. If candidate knows complex tools (e.g., Angular/Vue) but job needs React, give credit for ability to learn.
✓ **INFERRED SKILLS** = 0.5-1.0 credit. Infer standard competencies from tools. (e.g., Web Frameworks imply REST/HTTP knowledge; Cloud Providers imply DevOps basics).
✓ Soft skills = IGNORE (e.g., "communication", "teamwork") - Score these in Section 4.

EXAMPLES:
- Job needs: [React, Node.js, MongoDB, TypeScript, AWS, Docker] = 6 skills
- Candidate A has: [React, Node.js, MongoDB, JavaScript] = 3 full matches
  Score: 3/6 × 25 = 12.5 points
- Candidate B has: [Python, R, SQL, pandas] = 0 web dev skills
  Score: 0/6 × 25 = 0 points ❌

### 2. TITLE MATCH (30 points) - RESPONSIBILITIES OVER LABELS 🎯

Compare job title to candidate's "current_title", BUT prioritize the *nature of the work* (Responsibilities) over the exact label.
Titles are often misleading (e.g., "VP" at a 2-person startup vs "Associate" at Google).

⚠️ CRITICAL INSTRUCTION: Ignore the title if the description of responsibilities matches the seniority level required.
Focus on FUNCTIONAL EQUIVALENCE.

SCORING SCALE:
✓ 30 points: Functionally equivalent role (regardless of exact wording)
  Examples: "Full Stack Developer" ↔ "Software Engineer"
            "MTS / Member of Technical Staff" ↔ "Senior Engineer"
            "Consultant" ↔ "Developer" (if the project work is development)
            "Founder/CTO" (at small startup) ↔ "Senior Developer"
            "Associate" (at big tech) ↔ "Senior Developer" (at startup)

✓ 20-25 points: Related role, same domain or strong transferable skills
  Examples: "Backend Developer" for "Full Stack Developer"
            "Junior Developer" for "Developer"
            "Specialized Engineer" for "Generalist Role" (if core languages overlap)

✓ 10-15 points: Distant but relevant
  Examples: "Data Analyst" for "Data Scientist"
            "QA Engineer" for "Software Developer"

✓ 0 points: Completely different field
  Examples: "Teacher" for "Software Engineer" ❌
            "Air Traffic Controller" for "Developer" ❌
            "Student Projects" for "Senior Engineer" ❌

🚩 RED FLAG: If title score < 15 points, mark as "Risk" but do NOT disqualify if Skills Match is > 20 points.

### 3. EXPERIENCE LEVEL (20 points) 📅

Count ONLY relevant professional years (exclude: military, teaching unless tech-related, student projects).
Experience level is crucial - it shows capability and maturity.

**STABILITY ANALYSIS:**
- Do NOT penalize for short tenures (1-2 years) if there is clear progression in responsibility (Promotion) or valid reasons (Contract/Startup closure).
- Penalize only for "Job Hopping" without progression (e.g., 4 jobs in 2 years with same title).

SCORING LOGIC - PROXIMITY-BASED WITH SENIORITY ADJUSTMENT:

**Under-qualified:**
✓ < 50% of required years: 1-4 points (severely under-qualified)
✓ 50-75% of required years: 5-10 points (significantly under-qualified)
✓ 75-90% of required years: 11-15 points (moderately under-qualified)
✓ **90-99% of required years:** 18-19 points (Treat as effectively qualified - e.g., missing a few months is a PASS)

**Over-qualified:**

✓ Generally, treat extra experience as a BONUS, not a penalty.
✓ Only penalize if the gap is extreme (e.g., 15+ years for a Junior role), suggesting flight risk or boredom.

SCORING:
✓ Candidate has MORE years than required: 20 points (Full Score)
✓ EXCEPTION: If Candidate has >3x the required years for a JUNIOR role (e.g., 10 years for a 2-year role): 15 points.

**Key Principle:** Experience is an asset. Do not penalize candidates for being "too good" unless it's a clear mismatch (e.g. VP applying for Junior Dev).
**Key Principle:** Experience gaps have diminishing impact as seniority increases.
A candidate with 10 years applying for a 7-year senior role loses minimal points (17.6/20).
But a candidate with 7 years for a 4-year junior role loses more (12.8/20) due to over-qualification concerns.

EXAMPLES:
- Job requires: 5 years, Candidate: 5 years → 20 points ✓
- Job requires: 5 years, Candidate: 6 years → 20 points (within +1 year) ✓
- Job requires: 5 years, Candidate: 3 years → 10 points (under-qualified by 40%)
- Job requires: 4 years, Candidate: 7 years → 13 points (junior role, strict penalty)
- Job requires: 7 years, Candidate: 10 years → 18 points (senior role, gentle penalty) ✓✓
- Job requires: 10 years, Candidate: 15 years → 15 points (very senior, minimal penalty) ✓
- Job requires: 2 years, Candidate: 1 year → 4 points ❌ (severely under-qualified)
- Student with project work → 4 points ❌ (no professional experience)

### 4. CONTENT & CONTEXT MATCH (15 points) - SEMANTIC & CULTURE ANALYSIS 🔍

This section evaluates what the algorithm cannot see: semantic meaning, responsibilities alignment, and domain knowledge.
This is where the LLM adds unique value beyond algorithmic scoring.

**A. Keywords & Terminology (0-5 points)**
Does the resume use similar language/jargon OR imply understanding?
- Look for: methodologies (Agile, Scrum), tools (Git, Jira), concepts (microservices, CI/CD)
- **CRITICAL:** Do not penalize heavily for missing buzzwords like "Microservices" or "REST" if the candidate has built complex web applications where these are standard. Look for the *substance* of the work.

**B. Responsibilities Alignment (0-5 points)**
Did the candidate perform similar tasks/projects?
- Compare job duties to resume accomplishments
- Example:
  * Job: "Build scalable REST APIs for e-commerce platform"
  * Resume: "Developed payment APIs handling 10K requests/day" → 5 pts ✓
  * Resume: "Built internal tools for data visualization" → 2 pts

**C. Soft Skills & Culture Fit (0-5 points)**
- **Leadership Potential:** Mentoring, leading teams, ownership.
- **Team Player:** Collaboration, cross-functional work.
- **Tech Agility:** Evidence of learning new stacks quickly.
- **Domain:** Relevant industry experience (fintech, healthcare, etc.).

**Scoring Guidelines:**
- High match (12-15 pts): Resume mirrors job language, clear responsibility overlap, relevant domain
- Medium match (7-11 pts): Some keyword overlap, partial responsibility alignment
- Low match (0-6 pts): Different terminology, unrelated responsibilities, no domain experience

EXAMPLES:

**High Content Match (14/15):**
```
Job: "Senior Backend Engineer for fintech startup - build payment processing APIs"
Resume: "Led backend team at payment company, developed fraud detection APIs, 99.9% uptime"
→ Keywords: payment ✓, APIs ✓, backend ✓, fintech terminology (6 pts)
→ Responsibilities: fraud detection = security/compliance, API development ✓ (6 pts)
→ Domain: fintech experience, startup mention (2 pts)
Total: 14/15 ✓✓
```

**Low Content Match (4/15):**
```
Job: "Frontend Developer for SaaS dashboard - React, data visualization"
Resume: "Built marketing landing pages with HTML/CSS, A/B testing campaigns"
→ Keywords: marketing ≠ SaaS product terminology (2 pts)
→ Responsibilities: landing pages ≠ complex interactive dashboards (1 pt)
→ Domain: no SaaS/product development experience (1 pt)
Total: 4/15 ❌
```
